# GOV.UK Tariff API Documentation

This is a microsite for providing documentation for the GOV.UK Tariff API.

The framework is provided by the [GOV.UK Tech Docs Template][tech-docs-template].

These docs are deployed to the following URLs:

- *Development* - https://api.dev.trade-tariff.service.gov.uk
- *Staging* - https://api.staging.trade-tariff.service.gov.uk
- *Production* - https://api.trade-tariff.service.gov.uk

## Updating content

HTML pages are in the [`/source`][source-dir] directory and are authored using Markdown or ERB templates. Make edits in a branch and open a pull request.

### HTML pages

To update page content (index, getting started, reference, etc.), edit the corresponding `.md` or `.md.erb` file under `source/`:

- `source/index.html.md.erb`
- `source/getting-started.html.md`
- `source/reference.html.md.erb`

### OpenAPI spec (`source/openapi.yaml`)

`source/openapi.yaml` is the v2 API specification. It is published at `/openapi.yaml` and used to build `llms-full.txt` at site build time.

**Do not edit this file directly.** It is synced automatically from [`trade-tariff-backend/swagger/v2/swagger.json`][backend-swagger] by the [Update OpenAPI spec][update-workflow] workflow, which runs every Monday and opens a pull request if the spec has changed. You can also trigger it manually via `workflow_dispatch`.

To update the spec, make changes in the backend repo. They will be picked up on the next weekly run (or you can trigger the workflow manually).

## Running documentation locally

### Prerequisites

- Ruby (version in `.ruby-version`)
- Graphviz (`brew install graphviz` or equivalent)

### Install dependencies

```
make requirements
```

### Preview changes

```
make serve
```

The server runs at [http://localhost:4567](http://localhost:4567). HTML pages reload automatically on save. Restart the server after changes to ERB templates that load data files (e.g. `llms-full.txt.erb`).

### VS Code Dev Container

Open the folder in VS Code, then run **Dev-Containers: Open Folder in Container** from the command palette. The container builds and starts the server automatically at <http://localhost:4567>.

## Layout overrides (`govuk_tech_docs`)

This site overrides the Tech Docs gem's `_header.erb` and `_footer.erb` under `source/layouts/`. Header links and footer URLs are configured in `config/tech-docs.yml`.

After `bundle update govuk_tech_docs`, run `bin/tech-docs-layout-upstream-hint` — it prints the upstream GitHub paths for your gem version so you can merge any gem changes into those two files.

## Publishing documentation

### Development

Any changes pushed to GitHub on a branch (e.g. a pull request) are deployed automatically to the [development URL](https://api.dev.trade-tariff.service.gov.uk).

### Production

Any changes merged to `main` are deployed automatically to [api.trade-tariff.service.gov.uk](https://api.trade-tariff.service.gov.uk).
